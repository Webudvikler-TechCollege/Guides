# Prisma CLI – Simpel Guide

## Opsætning og init
| Kommando | Beskrivelse |
|-----------|-------------|
| `npx prisma init` | Opretter en ny `prisma/` mappe med `schema.prisma` og `.env`. |
| `npx prisma generate` | Genererer Prisma Client ud fra dit schema (kør efter ændringer). |

---

## Migrationer (databaseændringer)
| Kommando | Beskrivelse |
|-----------|-------------|
| `npx prisma migrate dev --name <navn>` | Opretter og kører en ny migration i udvikling. |
| `npx prisma migrate reset` | Sletter databasen og genskaber den fra migrationer. |
| `npx prisma migrate deploy` | Kører alle migrationer på en produktionsdatabase. |
| `npx prisma migrate status` | Viser status for migrationer (kørt/mangler/afventer). |

---

## Database
| Kommando | Beskrivelse |
|-----------|-------------|
| `npx prisma db push` | Skubber schema direkte til databasen (uden migrationer) – godt i tidlig udvikling. |
| `npx prisma db pull` | Opdaterer dit `schema.prisma` ud fra en eksisterende database. |
| `npx prisma db seed` | Kører din seed-fil (defineret i `package.json` eller `prisma/schema.prisma`). |

---

## Visualisering
| Kommando | Beskrivelse |
|-----------|-------------|
| `npx prisma studio` | Åbner et webinterface til at se og redigere data i databasen. |

---

## Eksempler på workflow

### Første opsætning
```bash
npx prisma init
```

### Efter schema-ændringer
```
npx prisma migrate dev --name added_user_table
```
### Tving schema til at matche databasen (uden migration)
```
npx prisma db push
```

### Inspicér og redigér data
```
npx prisma studio
```

### Kør seed-skript (hvis det findes)
```
npx prisma db seed
```
